# 工程造价知识系统实施方案

## 1. 目标

建立一套长期可运行的工程造价知识系统，把文档、视频、聊天记录、图片和系统审核结果，沉淀为可追溯、可搜索、可维护的 Markdown Wiki，并在 Obsidian 中进行浏览、修订和验收。

## 2. 角色分工

- OpenClaw：统一入口、任务编排、调度、提醒
- auto-quota / JARVIS：业务执行、审核、正式知识层
- knowledge_staging：候选知识缓冲层
- knowledge_wiki：中间 Wiki 编译层
- Obsidian：展示和人工修订层
- Codex：脚本、接口、同步、lint、自动化实现

## 3. 总体流程

```text
原始资料
  -> file-intake
  -> classify / parse / route
  -> source pack
  -> staging / wiki compiler
  -> knowledge_wiki
  -> Obsidian 浏览与修订
  -> 人工确认
  -> 正式知识层
```

## 4. 统一入口

### 4.1 文件入口

- OpenClaw：`/api/openclaw/file-intake/upload`
- 统一文件入口：`/api/file-intake/upload`

### 4.2 审核学习入口

- `review-confirm -> knowledge_staging -> knowledge_wiki`

### 4.3 历史资料入口

- 批处理脚本统一调用 `upload / classify / parse / route`
- 不允许让 LLM 直接扫描整个硬盘

## 5. 学习方法

LLM 不做模型训练，而做持续维护 Wiki。

固定循环：

1. ingest：把原始资料转成 source pack
2. compile：用 source pack 更新已有页面
3. link：补双链和索引
4. lint：检查缺陷和冲突
5. promote：人工确认后回流正式知识层

## 6. Source Pack 规则

Source Pack 是 LLM 的直接学习对象，最小结构如下：

```json
{
  "source_id": "",
  "source_kind": "doc|video|chat|image|system",
  "title": "",
  "summary": "",
  "full_text_path": "",
  "evidence_refs": [],
  "province": "",
  "specialty": "",
  "tags": [],
  "created_at": "",
  "confidence": 0
}
```

要求：

- 文档必须先提取正文
- 视频必须先转 transcript
- 图片必须先 OCR
- 聊天必须先清洗结论
- 系统审核结果可直接转 source pack

## 7. 搜索体系

搜索分三层，不能混用：

### 7.1 业务搜索

- 定额、候选、价格参考
- 服务业务执行，不服务知识沉淀

### 7.2 Wiki 搜索

- 规则、案例、方法、概念
- 小规模先用 `index.md + Obsidian 搜索`
- 后续必要时接 `qmd`

### 7.3 证据搜索

- source pack
- OCR
- transcript
- review 记录
- staging 记录

## 8. Wiki 目录

```text
  10-规则

  50-审核沉淀
  60-视频笔记
  70-待处理
  80-日报周报
  99-索引
```

## 9. 关键原则

- Obsidian 不是数据库
- 所有知识页必须带 `source_refs`
- 不确定内容只能进入 `draft/review`
- 正式规则页必须人工确认后晋升
- 优先更新旧页，避免重复建页

## 10. 分阶段计划

### P0 方案定版

交付：

- 架构文档
- source pack 规范
- wiki 页面规范
- 分阶段验收标准
- `AGENTS.md`
- `index.md`
- `log.md`

验收：

- 结构清晰
- 文档完整
- 目录可用

### P1 最小闭环：staging -> wiki -> Obsidian

交付：

- `export_staging_to_wiki.py`
- `sync_wiki_to_obsidian.ps1`
- 至少 3 个页面样例

验收：

- staging 内容可导出为 Markdown
- Obsidian 中可见
- 页面 frontmatter 完整

### P2 通用资料入口：文档 / 聊天 / 图片

交付：

- `ingest_document.py`
- `ingest_chat.py`
- `ingest_image.py`
- `compile_sources_to_wiki.py`

验收：

- 每类资料至少 2 个样例跑通
- 能从 Wiki 页面回溯到原始资料

### P3 视频入口

交付：

- `ingest_video.py`
- transcript 样例
- video source page 样例

验收：

- 至少 2 个视频跑通
- 可回溯 transcript 和时间轴

### P4 搜索层

交付：

- wiki 搜索
- 证据搜索
- 搜索说明文档

验收：

- 三层搜索边界清晰
- 3 类示例查询可跑通

### P5 治理与晋升

交付：

- `lint_wiki.py`
- `import_wiki_promotions.py`
- 治理文档

验收：

- lint 可出报告
- 至少 3 条页面能完成人工确认后回流
