# 阶段L2-a Findings — 建立固定Benchmark + 采集基线

## 日期
2026-02-23

## 发现清单

### F1: 缺少固定benchmark数据集（P1）
- **严重级别**: P1
- **描述**: 系统缺少固定的评测数据集，每次改进效果无法量化对比
- **影响**: 无法判断代码修改是"变好还是变差"，所有L2后续改进工作无法验证
- **复现方式**: 无 `tests/benchmark_baseline.json` 文件

### F2: 缺少脏数据鲁棒性测试（P2）
- **严重级别**: P2
- **描述**: 系统未针对缺字段、单位异常、描述噪声等脏数据做专项测试
- **影响**: 真实清单中常有脏数据，不测试则无法评估系统鲁棒性
- **复现方式**: 无 `tests/fixtures/dirty_data_sample.xlsx` 文件

### F3: 缺少benchmark回归测试（P2）
- **严重级别**: P2
- **描述**: 即使有基线数据，也缺少自动化的回归测试机制
- **影响**: 开发者改代码后可能忘记手动检查，导致退化被漏过
- **复现方式**: 无 `tests/test_benchmark_regression.py` 文件

### F4: B4脏数据有1条被过滤（观察项）
- **严重级别**: 观察
- **描述**: B4脏数据样本有20条，但只读出19条。第1条"名称为空"的清单项被 bill_reader 过滤掉了
- **原因**: `bill_reader._read_sheet()` 对名称为空的行判定为非清单行
- **影响**: 不影响评测有效性（系统对空名称清单不处理是合理行为）
- **后续**: 无需修复，仅记录
