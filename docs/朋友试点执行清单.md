# 朋友试点执行清单（V1）

更新时间：2026-02-22

## 1. 试点目标（2周）

- 验证：系统在真实项目中的可用性与准确性。
- 验证：朋友是否能按要求回传高质量人工结果。
- 输出：是否扩大开放范围的决策依据。

## 2. 试点范围

- 白名单人数：3-5 人。
- 每人每周任务上限：3 个项目。
- 只允许安装专业清单（先收敛场景）。

## 3. 账号与权限规则

- 账号实名到人，不允许共享。
- 免费试点必须回传人工最终结果。
- 超过回传时限（7天）自动暂停账号。

## 4. 标准流程

1. 朋友提交清单（生成 `task_id`）。
2. 系统输出自动匹配结果。
3. 朋友在结果上人工修订。
4. 朋友回传“人工最终版”（必须带 `task_id`）。
5. 你方自动做差异分析并打分。
6. 达标继续试点；不达标限额或暂停。

## 5. 回传数据最小字段（必填）

- `task_id`
- `user_id`
- `bill_row_key`（建议：sheet+序号+清单编码）
- `final_quota_id`
- `final_quota_name`
- `final_unit`
- `is_modified`（0/1）
- `modify_reason`（如有修改必须填）
- `submitted_at`

## 6. 质量评分规则（周评估）

- 行对齐率 >= 98%
- 必填完整率 >= 95%
- 有修改项中“可解释修改”占比 >= 80%
- 垃圾回传（乱填/批量复制）= 0 次

处理策略：
- 全部达标：继续免费试点。
- 1项不达标：降为限额。
- 2项及以上不达标：暂停账号。

## 7. 运营看板（每周统计）

- Top1命中率
- 人工改动率
- 自动纠错采纳率
- 失败率
- 降级占比（agent/vector/reranker）

## 8. 扩容闸门（从5人扩到20人）

同时满足以下条件才扩容：

- 连续2周失败率 <= 1%
- 连续2周降级占比 <= 20%
- Top1命中率 >= 85%
- 回传达标账号占比 >= 80%

## 9. 给朋友的通知模板（可直接发）

```
本次是白名单试点（2周），每周最多3个项目。
你可以免费使用，但需要在7天内回传人工最终版结果（带task_id）。
若未回传或回传质量不达标，账号会被限额或暂停。
本次仅用于验证效果和优化算法，感谢配合。
```
