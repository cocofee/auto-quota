# 阶段L2-b Findings — 参数提取率提升

## 日期
2026-02-23

## 发现清单

### F1: 管材代号DN与给排水DN需要区分（P1）
- **严重级别**: P1（已修复）
- **描述**: 将电气管材代号（SC20、PC25）提取为标准 `dn` 字段，导致 param_validator 误罚，B2绿率暴跌16.6pp
- **根因**: 电气配管定额不按管径分档，定额名称中没有DN参数。param_validator 在"清单有DN但定额无DN"时给0.5分，导致置信度从95降到47
- **修复**: 管材代号管径存为独立字段 `conduit_dn`（不在 TIER_PARAMS 中），不参与参数验证

### F2: param_validator 的"定额无参数"评分过低（P2）
- **严重级别**: P2（已修复）
- **描述**: 当清单提取到参数但定额没有对应参数时，评分为0.5（confidence=47，深黄）
- **分析**: 定额不含某参数通常意味着该定额是"通用型"（不按该参数分档），不代表匹配错误。例如"管道试压"定额不分DN，但试压定额匹配试压清单是正确的
- **修复**: 评分从0.5调到0.9（confidence=85，刚好绿色门槛），覆盖DN/截面/容量/回路/电流5项

### F3: 导线型号截面提取已支持（观察项）
- **严重级别**: 观察
- **描述**: BV-2.5、BYJ-4、WDZN-BYJ-4 等导线型号现在可以正确提取截面积
- **效果**: B1给排水绿率从80%提升到90%（+10pp），B3配套楼从94.7%提升到99%（+4.2pp）
- **原因**: 导线截面提取让 param_validator 能够正确进行截面匹配验证，将之前"定额有截面但清单未指定"（score=0.6）的项变为"截面匹配"（score≈1.0）

### F4: "直径"关键词和"X路"回路匹配（观察项）
- **严重级别**: 观察
- **描述**: 补充了"直径150"→DN150 的关键词识别，和"24路"→回路数24 的匹配
- **影响**: 对 B1 给排水数据有正向影响
