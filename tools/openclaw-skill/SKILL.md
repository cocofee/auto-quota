# Auto Quota 自动套定额 Skill v5.0

## 一句话

两种模式，OpenClaw只做简单确定的事，复杂判断交给Claude Code：
- **监控模式**：扫描微信目录的Excel -> 提交匹配 -> 确认绿灯 -> 经验库升级 -> 发报告
- **批量模式**：从本地jarvis目录按省份挑文件 -> 逐个跑 -> 每5个暂停汇总

## v5.0 分工原则

| 谁 | 做什么 | 为什么 |
|----|--------|--------|
| **OpenClaw** | 绿灯确认 + 经验库候选层升级 + 红灯搜索报告 | 简单规则，不需要判断 |
| **Claude Code** | 黄灯AI审核 + 经验库权威层降级 + 改算法 | 复杂判断，需要造价语义理解 |

**OpenClaw不做的事**：黄灯审核、经验库降级、自动纠正。这些全部通过报告交给Claude Code处理。

---

## 铁律（最高优先级）

| 规则 | 说明 |
|------|------|
| **禁止自动处理！** | 发现新文件只飞书通知用户，未经用户明确指示不做任何处理 |
| **用户说停立刻停！** | 收到停止/取消/别跑了等指令，立刻中止 |
| **报告必须完整！** | 标题+统计+经验库操作+待人工明细+问题汇总 |
| **一次一个任务** | 一个跑完再下一个 |
| **禁止自动纠正** | 不能改成别的定额 |

## OpenClaw处理流程（v5.0核心）

### 处理一个文件的完整步骤

```
1. 提交匹配 -> 轮询等完成
2. 拿全部结果
3. 分拣：绿灯 / 黄灯 / 红灯 / 措施项
4. 措施项 -> 批量确认+跳过
5. 绿灯(>=85%) -> 批量确认 -> 搜经验库升级候选层
6. 红灯(<60%) -> 去重后搜定额库诊断 -> 写报告
7. 黄灯(60-84%) -> 不处理，只写入报告交给Claude Code
8. 导出Excel
9. 发飞书报告
```

### Step 5 详解：绿灯确认 + 经验库升级

```
绿灯项(>=85%)
  -> POST批量确认（1次API调用）
  -> 收集所有绿灯的清单名，去重
  -> 对每个唯一清单名搜经验库：
     GET /api/admin/experience/search?q=清单名&province=省份
  -> 搜到候选层记录 且 定额编号一致
     -> 升级到权威层: POST /api/admin/experience/{id}/promote
  -> 搜到候选层记录 但 定额编号不一致
     -> 不动，报告标注"经验库冲突"
  -> 搜不到 -> 跳过
```

**升级的判断逻辑很简单**：绿灯(系统有把握) + 经验库有同款(候选层) + 定额编号一致 = 可以升级。三个条件缺一不可。

### Step 6 详解：红灯诊断

```
红灯项(<60%)
  -> 收集清单名去重（同名不同DN的只搜1次）
  -> 对每个唯一清单名调智能搜索：
     GET /api/quota-search/smart?name=清单名&province=省份
     （用curl！不用Python urllib！URL要短！）
  -> 搜到结果 -> 写入报告：原匹配 vs 搜索候选
  -> 搜不到 -> 报告标注[冷]或[跨]
```

**红灯不做经验库操作**——降级是危险操作，留给Claude Code判断。只把红灯信息完整地写进报告。

### Step 7 详解：黄灯直接进报告

```
黄灯项(60-84%)
  -> 不确认、不搜索、不操作经验库
  -> 直接写入报告的"待Claude Code审核"区域
  -> 报告里写清楚：清单名、匹配定额、置信度、描述特征
```

**黄灯是OpenClaw和Claude Code的交接点**——OpenClaw把信息整理好，Claude Code拿到报告后做5项检查+经验库降级。

## Rules

1. **>=85%绿灯批量确认** + 搜经验库升级候选层同款
2. **60-84%黄灯不处理**，只写报告交Claude Code审核
3. **<60%红灯搜索诊断**，写报告，不确认不纠正不降级
4. **经验库升级条件**：绿灯 + 候选层有同款 + 定额编号一致，三条全满足才升级
5. **经验库降级由Claude Code做**，OpenClaw不做降级操作
6. **禁止自动纠正**：不能改成别的定额
7. 每个文件处理完发飞书报告（含经验库升级统计+黄灯明细）
8. 每天22:00发日报汇总
9. 同一文件不重复处理
10. **省份用完整名称**：必须用 /api/quota-search/provinces 返回的全称
11. **搜索API用curl**：不用Python urllib（会被Cloudflare拦403）
12. **搜索URL要短**：只传清单名称（<=50字符）

## 系统级问题观察（每次必做！）

|----------|------|

| **匹配耗时异常** | 等待时间变长 -> 系统性能问题 |

## 踩过的坑

- **搜索API用curl** — Python urllib被Cloudflare拦403
- **搜索URL要短** — 只传清单名（<=50字符）
- **监控目录** — `/lzcapp/document/...` 不是 `/lzcapp/data/...`
- **飞书不发canvas链接** — 双下划线被吃掉
- **确认=进经验库** — 错的会教坏系统，宁漏不错
- **压缩包中文乱码** — 用 `unzip -O GBK`
- **重复文件** — (2)后缀副本只取不带后缀的
- **GBQ/Qdg格式** — 不支持，跳过
- **匹配引擎可能不稳定** — 失败的记录下来，不反复重试
