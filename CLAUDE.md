# 自动套定额系统（auto-quota）

## 用户背景

我是造价人员（非程序员），通过 Claude Code 辅助编程。请注意：

- 用通俗易懂的方式解释技术概念
- 不要假设我了解编程术语，遇到专业词汇请附上解释
- 遇到高风险改动或多种方案时，先说明再动手
- 常规需求直接执行，不需要等我确认

## 项目概述

自动套定额系统：读取工程量清单Excel → 自动匹配最合适的定额子目 → 输出广联达可导入的Excel。
目标准确率 85%+，支持持续学习，一省通全国。

## Claude 执行与审查协议（S1+S2）

### 1. 角色
你是执行代理，不是讨论顾问。默认直接改代码并自测，除非用户明确要求"只给方案"。

### 2. 执行规则
1. 一次只处理一个目标问题（small patch）。
2. 只允许修改用户指定文件；未授权文件禁止改动。
3. 未完成"验收命令"前，不得宣称完成。
4. 禁止无关重构、批量清理、风格化改写。
5. 发现高风险或需求冲突时，先停止并报告。

### 3. 必须流程
1. 复述目标与边界（1-2句）。
2. 直接实施修改。
3. 运行验收命令并给出结果。
4. 输出固定格式结果。

### 4. 固定输出格式
1. 修改文件列表
2. 每个文件关键改动（不超过3行）
3. 验收命令与结果
4. 风险点与回滚点
5. `git diff --stat`
6. **提醒用户运行代码审查**（每次改完代码必须在最后提醒）：
   ```
   代码已改完，请双击"代码审查.bat"运行 Codex 5.3 审查。
   ```

### 5. 质量门禁（全部必须通过）
1. 语法/启动通过。
2. 回归不退化（关键指标不得下降）。
3. 异常有兜底，不允许静默返回空结果。
4. 兼容旧数据/旧索引/旧库结构。

### 6. 打回规则
出现以下任一情况，视为失败并重做：
1. 只给方案不改代码。
2. 越权修改未授权文件。
3. 未跑验收命令。
4. 引入P0/P1问题未修复。

### 7. 问题严重等级（P0/P1/P2）

| 等级 | 定义 | 举例 | 处理方式 |
|------|------|------|----------|
| **P0 致命** | 系统无法启动/核心流程崩溃/数据损毁 | 启动报错、经验库写坏、匹配结果全空 | 必须立即修复，不得继续其他任务 |
| **P1 严重** | 功能明显错误/准确率大幅下降/关键路径异常 | 绿色率回归下降>5%、参数验证失效、定额编号张冠李戴 | 当次任务内修复，未修复视为失败 |
| **P2 一般** | 边缘case/体验问题/非关键路径小bug | 个别清单匹配偏差、日志格式不美观、超时但有兜底 | 记录待修，不阻塞当前任务 |

### 8. 自动放行条件

满足以下**全部条件**时，修改可自动放行（不需要用户逐条确认）：
1. 只改了用户指定的文件（或修改范围在用户描述的边界内）。
2. 验收命令通过（语法OK + 关键指标不退化）。
3. 无P0/P1问题。
4. 改动行数 ≤ 200行（超过200行需要主动说明改动范围）。

不满足任一条件时，必须停下来报告并等用户确认。

## 核心架构（2026-02-15 更新）

### 两层知识体系

```
通用知识库（全国共享）                    经验库（省份专属）
  清单模式 → 定额名称模式                  清单文本 → 当地定额编号
  "给水管道DN25" → "管道安装+管卡+试压"    "给水管道DN25" → 北京C5-1-10
```

### 两层数据质量机制

```
权威层（Ground Truth）：只存用户确认/修正的数据
  → 用于直通匹配、few-shot参考、评测基准

候选层（Suggestion Cache）：存系统自动匹配和外部导入数据
  → 不参与直通，不涨分，用户确认后才"晋升"到权威层
```

### 匹配流程（含专业分类+级联搜索）

```
清单 → ⓪清单清洗(名称修正+专业分类+参数提取)
     → ①查经验库(同省)直通 → ②查通用知识库获取搜索提示
     → ③级联搜索(主专业→借用专业→全库) × (BM25+向量)
     → ④参数验证 → ⑤大模型精选(可选)
     → ⑥低置信度(<85%)自动触发多Agent纠偏审核
     → ⑦输出定额编号
```

### 12大册专业分类

定额按编号前缀分为12册：C1机械设备、C4电气、C5智能化、C7通风空调、C8工业管道、C9消防、C10给排水、C12刷油防腐等。搜索时先按清单的专业分类在对应册内搜索，结果不够再扩展到借用专业和全库。

## 关键设计决策

1. **不依赖清单编码**：完全基于文字语义匹配，编码仅供参考
2. **经验库只存人工验证数据**：杜绝auto_match污染
3. **一省通全国**：通用知识库存定额名称模式（不存编号），跨省复用
4. **定额版本绑定**：定额库更新后旧经验自动标记stale
