# OB：搜索能力优先路线图
日期：2026-04-08

## 当前决策

本轮不让 OpenClaw 直接替代 Jarvis 做首轮定额匹配。

当前统一执行口径调整为：

1. 先提升首轮搜索能力。
2. 再提升排序与参数对齐能力。
3. 最后再用 OpenClaw 降低人工复核量。

## 原因

- OpenClaw 当前更适合作为复核器，而不是首轮主搜引擎。
- 目前主要损失并不只在二审，而是在首轮的召回不足、候选池不稳、top1/top2 分不开。
- 如果首轮候选池不够好，OpenClaw 只能在错误或不足的候选里做后置判断，收益有限。

## 本轮路线顺序

### 1. 先补诊断

- 按 `route / province / specialty / book` 做诊断统计。
- 单独盯住：
- `routing_miss`
- `keyword_miss`
- `insufficient_candidates`
- `small_score_gap`
- `low_param_score`
- `experience_review_rejected`

### 2. 再补召回

- 优先做 query rewrite、多路检索表达、材料/安装分流、跨册借册修正。
- 目标不是“换一个复核器”，而是让正确答案先稳定进入候选池。

### 3. 再补排序

- 重点拉开 top1/top2 gap。
- 重点强化参数、单位、材质、family/system 对齐。

### 4. 最后才补 OpenClaw 自动化

- OpenClaw 继续聚焦黄灯/红灯和高风险歧义样本。
- 不把它改造成首轮检索器。

## 两周执行口径

第一周：

1. 做诊断统计。
2. 做 query rewrite 和 route 修正。
3. 复跑评测，确认 `routing_miss / insufficient_candidates` 是否下降。

第二周：

1. 做候选池扩容。
2. 做排序和 confidence 校准。
3. 复跑评测，确认 `small_score_gap / low_param_score` 是否下降。
